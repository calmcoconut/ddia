Interleaved Scenario Challenge

Mixes all topics · Builds discrimination · Forces transfer · ~10 min

Each scenario requires you to combine knowledge from multiple sections of Chapter 5. This deliberate mixing forces your brain to select the right framework — the hallmark of expert-level understanding.

Scenario 1 of 4

📦 Financial Ledger Microservices

A fintech company runs a global ledger system using a microservice architecture. The core transaction service publishes 'TransactionCreated' events to Apache Kafka. The audit service consumes these events and writes them to an immutable S3 storage archive (Parquet/Avro). The analytics service reads them to score fraud risk. The ledger team wants to roll out a new schema containing an optional regulatory_compliance_id string field.

Scenario 2 of 4

🤖 Smart Factory Durable Execution

You are building an IoT automation workflow for a smart factory. The workflow orchestrates:

  • Checking the safety status of assembly robots (via gRPC query to service A)
  • Issuing a start command to the robots (via POST request to service B)
  • Generating a random authorization code (to pass to the safety monitor)
  • Writing a confirmation log to a database

You choose a durable execution framework (like Temporal) to manage this workflow.

Scenario 3 of 4

🎮 Distributed Game Lobby Actors

A multiplayer gaming company uses a distributed actor framework (like Akka or Orleans) to manage game rooms. Each game room is an actor. Players connect to the lobby service, which redirects them to game nodes. The nodes communicate by passing messages containing game state (player position, health, inventory). The developers want to roll out an upgrade to the game node software using a rolling upgrade strategy.

Scenario 4 of 4

⚖️ Trade-Off Tribunal

You must make a ruling on each trade-off. There is no "right answer" — only well-reasoned arguments.

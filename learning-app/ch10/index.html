03

Interleaved Scenario Challenge

Mixes all topics · Builds discrimination · Forces transfer · ~10 min

Each scenario requires you to combine knowledge from multiple sections of Chapter 10. This deliberate mixing forces your brain to select the right framework — the hallmark of expert-level understanding.

Scenario 1 of 4

🏥 Distributed Medical Registry

A global healthcare database stores patient allergy records. Consider these constraints:

  • Doctors in different clinics globally must read and write allergy records.
  • A doctor in Europe updates a patient's allergy profile to record a severe Penicillin allergy.
  • Seconds later, a doctor in Asia administers a medication to the same patient.
  • The network between Europe and Asia is completely partitioned.
Scenario 2 of 4

💳 High-Throughput Bank Ledger

A bank is designing a ledger system for peer-to-peer transfers. Consider these constraints:

  • The system must process 5,000 transactions per second (TPS) globally.
  • It must enforce that account balances never drop below zero (a hard constraint).
  • The database cluster consists of 5 nodes geographically distributed.
Scenario 3 of 4

🤖 Cluster Management in a Cloud Platform

A container orchestration platform (like Kubernetes) manages 10,000 physical servers. Consider these constraints:

  • A master controller must coordinate worker nodes and allocate workloads.
  • It elects a single active leader controller to prevent split-brain conflicts.
  • It relies on etcd (a coordination service running Raft) for leader election.
Scenario 4 of 4

🔀 Trade-Off Tribunal

You must make a ruling on each trade-off. There is no "right answer" — only well-reasoned arguments.

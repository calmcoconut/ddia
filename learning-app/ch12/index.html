Scenario 1 of 4

💳 Financial Transaction Fraud Pipeline

A payment processing platform handles transaction events at 10,000 req/sec. It needs to:

  • Flag fraudulent events within 100ms by comparing the current transaction against the user's 2-hour sliding transaction history.
  • Join transaction events with a database table of user profile details (e.g., home address).
  • Ensure all transactions are written to a regulatory audit log without any data loss.
Scenario 2 of 4

🚚 IoT Fleet Location Tracker

A logistics company tracks 5,000 trucks via cell networks. The trucks:

  • Send periodic GPS coordinates and speed readings.
  • Travel through tunnels or remote zones, buffering events on the device for up to 30 minutes.
  • Trigger alerts on a dashboard when a truck has been speeding for over 5 minutes continuously.

  • Expose search query logs to an indexer to update search autosuggestions.
  • Mirror the PostgreSQL product inventory changes into a search index (Elasticsearch).
